# Лабораторная работа 1

## Тема: Алгоритмический мини-пакет (duck typing и контрактное программирование)

* **Цель:** освоить duck typing и контрактное программирование на примере источников задач - реализовать подсистему приёма задач в платформе обработки задач, где задачи могут поступать из различных независимых источников, объединённых единым поведенческим контрактом.
* **Вариант:** три источника задач: файл JSONL, текстовый файл, генератор, описание контракта через `typing.Protocol`, runtime-проверка соблюдения контракта, пользовательские исключения.
* **Используемые библиотеки:** `dataclasses`, `pathlib`, `json`, `random`, стандартная библиотека Python.

## Ограничения и допущения

- Запрещено использование общего базового класса для источников задач.
- Контракт описан с помощью `typing.Protocol` с декоратором `@runtime_checkable`.
- Задача — структура данных с двумя обязательными полями: `id` (целое) и `payload` (произвольные данные).
- `FileSource` читает задачи из текстового файла, где каждая строка имеет формат `<id> <payload>`.
- `JsonSource` читает задачи из JSONL-файла.
- `GeneratorSource` программно генерирует заданное количество задач со случайными payload.
- При нарушении контракта функция `receive_tasks` выбрасывает `InvalidTaskSourceError`.
- При некорректном формате данных в файлах выбрасываются соответствующие исключения.

## Реализованный функционал

### Контракт и структура задачи

#### Класс `Task`

- `Task(id, payload)` - класс, представляющий задачу.
  - `id: int` - уникальный целочисленный идентификатор задачи.
  - `payload: str` - произвольные данные задачи.

#### Протокол `TaskSource`

- `TaskSource` - `typing.Protocol` с декоратором `@runtime_checkable`, описывающий контракт источника задач.
  - Единственный обязательный метод: `get_tasks() -> list[Task]`.
  - Позволяет добавлять новые источники без изменения существующего кода.

### Функция приёма задач

- `receive_tasks(task_source: TaskSource) -> list[Task]` - принимает объект-источник, проверяет соответствие контракту через `isinstance(task_source, TaskSource)` и возвращает список задач.
  - Если объект не реализует контракт - выбрасывает `InvalidTaskSourceError`.

### Источники задач

#### `JsonSource`

- `JsonSource(path)` - источник задач из JSONL-файла.
  - Каждая строка файла - JSON-объект с полями `id` и `payload`.
  - Если файл не найден - выбрасывает `TaskFileNotFoundError`.
  - Если строка не является валидным JSON - выбрасывает `InvalidTaskFormatError`.
  - Если в JSON-объекте отсутствует поле `id` или `payload` - выбрасывает `MissingTaskFieldError`.
  - Если `id` не приводится к `int` - выбрасывает `InvalidTaskFormatError`.
  - Пустые строки в файле игнорируются.

#### `FileSource`

- `FileSource(file_path)` - источник задач из текстового файла.
  - Каждая строка файла: `<id> <payload>` (разделитель - первый пробел).
  - Если файл не найден - выбрасывает `TaskFileNotFoundError`.
  - Если строка не содержит двух частей или `id` не является числом - выбрасывает `InvalidTaskFormatError`.
  - Пустые строки игнорируются.

#### `GeneratorSource`

- `GeneratorSource(count)` - источник задач, генерирующий `count` задач программно.
  - `id` задачи.
  - `payload` - случайно выбранная строка из встроенного набора примеров.

#### `WrongSource`

- `WrongSource` - некорректный источник для демонстрации runtime-проверки.
  - Реализует метод `get_task()` вместо `get_tasks()`, то есть не соответствует протоколу `TaskSource`.
  - При передаче в `receive_tasks` будет выброшен `InvalidTaskSourceError`.

### CLI-интерфейс

- `main()` - интерактивный цикл выбора источника задач через консоль.
  - Пользователь выбирает источник вводом числа от 0 до 4.
  - После выбора источника выводится список полученных задач с их `id` и `payload`.
  - При ошибке выводится сообщение об ошибке без завершения программы.
  - Ввод `0` завершает программу.


## Установка и запуск

```bash
git clone https://github.com/Pewter71/MAI-python-lab6.git
cd MAI-python-lab6
```

Установка зависимостей:

```bash
uv sync
source .venv/bin/activate
```

Запуск интерактивного терминала:

```bash
python -m src.main
```


Запуск тестов:

```bash
pytest -q
```
